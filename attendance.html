Attendance System

This attendance system lets students mark themselves present in class. Before a student can mark him/herself present, he/she will have to pass two tests. The first test regards an IP Address check. This test checks whether or not the student is in school or not. The second test has to do with a random number that will be provided by the professor during class. If the number entered by the student matches the number generated by the professor, then the student will be marked present.

    Users will have to enter their credentials and select an appropriate radio button to sign in:
  • Professors will have to select a Faculty option after entering their login information
  • Similarly, students will have to select a student option after entering their login information
  • In case of picking wrong option, user will be promoted to error message. Click on the picture to check that error message.
    • User will be directed to the appropriate page after a successful login. Login consider to be successful if a user successfully passes the authentication process. Authentication process include following requirements:
  • User login and password will be read from the table called Professors or Students which are shown prior to above picture.
  • The input login should NOT be case sensitive, but the password should be case sensitive.
  • If the login or password is empty, corresponding error messages will be displayed and the program will exit immediately without accessing the database.
  • If both login and password are not empty, the program will access the table Professors or Students to verify the login and password.
  • If the login or username does not exist in the database, an error message “Login XXXX doesn’t exist in the database” will display and program will be terminated.
  • Similarly, If the login exists in the database, but password doesn’t match the record in database, an error message “login xxxx exists, but password not matches.” will display and program will be terminated.
  • If the login and password match the records in the database, “New php” file will display appropriate page including that user relevant information
  • Once a user is logged in, their corresponding home page will appear. In this page, the user can see and select their courses that they are enrolled in/ teach. Here, you can see the hours of each class and the days in which they are held.
    • What will happen if the user is a professor, and they select a course? If the user is a professor, and they select a course, the information of their students will appear. This information includes their name, email address, and the number of times that they attended the course. In addition to this, there is a “Generate Random Number” button, which will create a random number that the students must match in order to be marked as present for the day.
    What will happen if the user is a students, and they select a course?
    If the user is a student and they select one of their courses, they are redirected to the student attendance page. Here, they are asked to put in the random number that was generated by the professor at the beginning of the class, as well as to be connected to the school’s internet services. Once they click the submit button after correctly inputting the random number, they will receive a corresponding message telling them that whether they are to be marked present, or not. Moreover, students are not allowed to mark themselves present more than once for each class in a day.
    User will be given alert message after signing out from the portal and leads to the sign in page from there automatically after user confirm the message.
