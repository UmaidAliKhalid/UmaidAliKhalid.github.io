Banking System

Banking system which enable the users to login into their bank accounts to check their latest balance, track all of their transactions, deposit or withdrawal money, delete the history of transactions and update the side notes simultaneously.

    Main page for users to login into their accounts. This page contains following options:
  • Link to display the sensitive information of customers for testing purpose
  • Two textboxes for customers to enter login ID and password, and a submit button
  • Moreover, POST method is used to implement the HTML FORM.
    • These are all the users exist in database and information related to them. Click on the picture to check constraints and fields of the table.
    User will be directed to this page after a successful login. Login consider to be successful if a user successfully passes the authentication process. Authentication process include following requirements:
  • Customer login and password will be read from the table called Customers which is shown prior to this picture.
  • The input login should NOT be case sensitive, but the password should be case sensitive.
  • If the login or password is empty, corresponding error messages will be displayed and the program will exit immediately without accessing the database.
  • If both login and password are not empty, the program will access the table Customers to verify the login and password.
  • If the login or username does not exist in the database, an error message “Login XXXX doesn’t exist in the database” will display and program will be terminated.
  • Similarly, If the login exists in the database, but password doesn’t match the record in database, an error message “login xxxx exists, but password not matches.” will display and program will be terminated.
  • If the login and password match the records in the database, “login.php” file will display this page including that user relevant information such as name, DOB, current IP address, age, address, transactions, balance etc.
    • Let's talk about the other functions of this page such as logout, add transaction, display and update transaction and search box field:
  • Logout link will call a program to clean/expire the cookie.
  • All the 3 functions (search, add and update) will check if the cookie was set first. If the cookie is not set, programs should ask users to login first and exit without connecting to the database.
  • All the CGI programs are using the POST method, except the Search which will use GET method.
    • What does add transaction button perform?
  • login.php file would pass customer name to add_transaction.php file. Add transaction page would show the customer name and the balance at the top as shown in the figure.
  • Page would allow the customer to enter transaction code in textbox, select deposit or withdraw (no default) from two radio buttons, enter the amount in textbox, select source name in a dropdown list (no default), and a submit button to add the data into the Money table saved in database.
  • Program will retrieve all the source id and names first from Sources table, and will only display the source names in the dropdown list for the customer to select and will pass the source id to the next program insert_transaction.php file.
  • After customer clicks the Submit button, a program named “insert_transaction.php” should be called to insert the data into your Money table after verifying the input based on the following requirements.
      An error message would appear indicating the problem if the amount is empty, amount <=0, the type (deposit, withdraw) or a source is not selected.
      Program will show an error message indicating the problem if the balance is less than the withdraw amount. Program will retrieve and check the latest balance for the logged in customer from the Money table before inserting the withdraw transaction because the balance in the program might be outdated.
      An error message will pop up indicating the problem if the Same Code exists in the table.
      If there is no error, data would be inserted into the Money table. Program will retrieve customer id from the browser cookie and insert the customer id into the field cid of Money table. Negative amount would be inserted into the amount column of Money table if it is withdraw.
      If there is a data error, no data should be added into the Money table.

    Search function on the main page, it will call a program named “search.php” to do the following search functions after clicking on the “Search” button:
  • Program will use the keywords which is entered by customer to do pattern match on the Note columns for the logged in customer in Money table.
  • If the keyword is empty, an error message “no keyword entered” should be displayed and the program should not connect/access the database.
  • If the keyword is *, all the records (rows and cols or transactions) should be displayed.
  • If no record matched, “No record found with the search keyword: ….” will display. Additionally, no column headers should be displayed.
  • If records found, program will display all matched records in HTML TABLE tag format with headers.
    • Display and Update transaction:
  • If customer clicks on “Display and update transaction” link, a program named “display_update_transcation.php” should show all the customer’s transactions from Money table on webpage as shown in Figure.
  • The Note field is updateable and highlighted in yellow. All other columns are Non-updateable fields.
  • Delete column has the HTML checkbox for each record after the Note column.
  • If customer clicks on “Update transaction” button, a program named “update_transcation.php” should be called to do the followings:
      If a record’s delete checkbox is checked, that record should be deleted from Money table. A message “The Code xxxx has been deleted from the database.” should be displayed for each deleted record.
      If a record’s delete checkbox is not checked, but the Note was updated, program should update the Note column of the corresponding record in the Money table. A message “The Note for code xxxx has been updated in the database.” should be displayed for each updated record.
      Program can handle mixing delete and update operations. All the change and delete should be successfully done by ONE click, and other not changed records should not be updated.
      Message indicating the # of deleted records and the # of updated records will display after successful execution.
      Corresponding error message will display and indicate the Code of the record if an error happens during the delete or update.
