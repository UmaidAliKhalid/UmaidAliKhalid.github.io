Hospital Login Portal

Login portal is specifically designed and developed to ease lives of doctors and patients. This will minimize the load of paperwork for both doctors and patients since most of the work would be handled smoothly from Portfolio such as monitoring appointments, medications, patients, doctors etc.

    • Main page for users to login into their accounts. This page contains following options:
  • Links to display the sensitive information of doctors and patients for testing purpose
  • Radio buttons for corresponding users to select
  • Two textboxes for users to enter login ID and password, and a submit button
  • Moreover, POST method is used to implement the HTML FORM.
    • These are all the users exist in database and information related to them.
    Users will have to enter their credentials and select an appropriate radio button to sign in:
  • Doctors will have to select a doctor option after entering their login information
  • Similarly, patients will have to select a patient option after entering their login information
  • In case of picking wrong option, user will be promoted to error message. Click on the picture to check that error message.
    • User will be directed to the appropriate page after a successful login. Login consider to be successful if a user successfully passes the authentication process. Authentication process include following requirements:
  • User login and password will be read from the table called Patients or Doctors which are shown prior to above picture.
  • The input login should NOT be case sensitive, but the password should be case sensitive.
  • If the login or password is empty, corresponding error messages will be displayed and the program will exit immediately without accessing the database.
  • If both login and password are not empty, the program will access the table Patients or Doctors to verify the login and password.
  • If the login or username does not exist in the database, an error message “Login XXXX doesn’t exist in the database” will display and program will be terminated.
  • Similarly, If the login exists in the database, but password doesn’t match the record in database, an error message “login xxxx exists, but password not matches.” will display and program will be terminated.
  • If the login and password match the records in the database, “New php” file will display appropriate page including that user relevant information
    • What does Profile logo mean in both patients and Doctors page? Profile logo contains relevant information of the user such as username, ID, First Name, Last Name, Gender, DOB, Email Address, Blood Type and Address and Doctor name for patient user only.
